이 장의 내용

  • 자바가 거듭 변화하는 이유
  • 컴퓨팅 환경의 변화
  • 자바에 부여되는 시대적 변화 요구
  • 자바 8과 자바 9의 새로운 핵심 기능 소개
- 1996 Java Development Kit(JDK 1.0) 발표
- 책의 저자는 자바 역사를 통틀어 가장 큰 변화가 자바 8에서 일어났다고 적었음.내 주관적인 생각으로는 아마 람다식이나 스트림 사용함으로써 불필요한 소스를 생략하는 것 때문에 그렇게 생각하는것 같다.
- 멀티코어 CPU의 대중화 같은 하드웨어적인 변화도 자바 8에 영향을 미쳤다.
자바 8 이전의 버전은 대부분 코어 중 하나만을 사용하였다.(나머지 코어는 유휴 idle상태로 두거나, 운영체제나 바이러스 검사 프로그램과 프로세스 파워를 나눠서 사용)
  • JAVA1 :: 스레드와 락, 메모리 모델
  • JAVA5 :: 스레드 풀, 병렬 실행 켈렉션
  • JAVA7 :: 포크/조인 프레임워크
  • JAVA8
    ＊ 스트림 API ＊ 메서드에 코드를 전달하는 기법 ＊ 인터페이스의 디폴트 메서드
  • JAVA9 :: 리액티브 프로그래밍
